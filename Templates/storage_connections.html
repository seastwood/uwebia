External Drives

Connect to Google Drive (and, eventually, other cloud drives) so you can browse and import files into your asset library.

Google Drive · OAuth app credentials

Paste the client_id and client_secret from your Google Cloud project. You only need to do this once.

How to get these (≈15 minutes) ↓
  1. Go to Google Cloud Console and create a new project (or pick an existing one).
  2. Open APIs & Services → Library, search for Google Drive API, and click Enable.
  3. Open APIs & Services → OAuth consent screen, fill in the basic info, and add the scope https://www.googleapis.com/auth/drive.readonly.
  4. Open APIs & Services → Credentials, click Create Credentials → OAuth client ID, choose Web application.
  5. Under Authorized redirect URIs, add this exact URL:
    {{ google_redirect_uri }}
  6. Save, then copy the Client ID and Client Secret into the form below.

Local & network shares

For NFS or any pre-mounted share, set one or more allowlisted root paths below — then add a connection that points to one of them. For SMB you can either mount it at the OS level (and use Local Path) or connect to it directly with credentials.

Each path must already exist and be readable by the app process. Connections can only point at — or inside — one of these roots.
